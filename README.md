# Short Video Link Collector

一个面向 `YouTube Shorts` 和 `TikTok 账号主页视频列表` 的小型 Chrome 扩展。

它解决的核心问题是：

- 批量采集当前页面可见短视频封面链接
- 自动去重，避免重复复制
- 记录“上次采集到哪个封面”
- 导出 Excel 兼容表格，方便后续整理和交付

当前版本采用 **半自动采集** 模式：你负责在目标页面里手动滚动，扩展负责采集当前可见封面、记录断点并导出数据。

## 当前功能

- 识别当前 YouTube 频道 Shorts 页面
- 识别当前 TikTok 账号主页视频列表页面
- 一键采集当前视口内可见的视频卡片
- 在 YouTube 列表页每个视频卡片下方自动显示发布时间
- 按 `视频ID / 链接` 去重
- 保存当前页面的采集进度到浏览器本地存储
- 显示最近一次采集时间、累计数量、批次号
- 显示上次停留的最后一个封面标题与链接
- 导出 Excel 兼容 `.xls` 文件
- 导出 `.json` 备份文件
- 支持按页面 + 排序方式分别保存进度

## 快速开始

1. 打开 Chrome，进入扩展管理页：`chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择目录：`/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension`
5. 打开目标 YouTube `Shorts` 页面或 TikTok 账号主页视频列表
6. 点击扩展图标，执行“采集当前可见封面”
7. 需要导出时点击“导出 Excel 兼容表格”

更详细的操作步骤见：`/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/docs/使用说明.md`

## 断点续采逻辑

扩展会按以下维度隔离保存数据：

- 页面地址，例如频道 `Shorts` 页或 TikTok 主页
- 当前排序方式，例如“最新 / 最热 / 最早”

每次采集后，会记录：

- 当前累计采集数
- 当前批次号
- 最近一次采集时间
- 当前批次最后一个封面的标题、链接、缩略图
- 每条视频的发布时间，格式为 `YYYY-MM-DD`

下次打开同一个页面和排序时，可以直接看到上次停留的位置，然后继续滚动并采集。

## 已知限制

- 当前版本只适配半自动流程，需要手动滚动页面
- YouTube / TikTok 页面结构如果大改，可能需要更新 DOM 选择器
- 导出的 `.xls` 是 Excel 兼容格式，不是原生 `.xlsx`
- 当前以页面中“可见的视频卡片”为准，不抓取未渲染到页面里的内容
- TikTok 当前已支持链接采集与导出，但发布日期标签仍在适配中，页面上可能暂时看不到日期展示

## TikTok 日期策略

为了尽量降低页面风控风险，TikTok 页面默认不逐条访问每个视频详情页获取发布日期，而是按以下优先级取值：

1. 优先读取主页已渲染状态中的日期信息
2. 若页面未直接暴露日期，则根据视频 ID 还原发布日期

最终统一格式化为 `YYYY-MM-DD`。

当前状态说明：

- 采集与导出链路已经接入 TikTok
- 日期解析逻辑已接入
- 日期在 TikTok 宫格卡片下方的可视化展示仍需继续适配
