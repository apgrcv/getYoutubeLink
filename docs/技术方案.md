# 技术方案

## 1. 目标

围绕 `YouTube Shorts` 页面与 `TikTok 账号主页视频列表`，提供一个 **半自动批量采集链接工具**，满足以下需求：

- 批量获取当前页面可见视频封面链接
- 导出为 Excel 可打开的表格
- 记录采集进度，明确“已经采集到哪个封面”
- 支持下次继续采集

## 2. 为什么选 Chrome 扩展

相比纯 API 方案，这个项目更适合基于浏览器页面做采集，原因有三点：

1. 你要的是“页面上当前能看到的封面”，DOM 才是最准确的数据来源。
2. YouTube 和 TikTok 页面都是动态加载的，页面可见结果不一定和接口返回顺序完全一致。
3. 断点续采本质上是一个页面状态问题，扩展天然适合做本地状态保存。

## 3. 当前版本架构

### 3.1 content script

文件：`/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension/content.js`

职责：

- 识别当前页面所属平台与页面类型
- 读取当前排序标签
- 扫描当前视口内可见的视频卡片链接
- 找到对应卡片容器
- 读取标题、链接、视频 ID、缩略图、屏内顺序、发布时间
- 自动把发布时间插入到列表页每个卡片标题下方
- 将结果返回给 popup

当前实现采用“通用主流程 + 站点适配器”结构：

- `YouTube` 适配器负责 `Shorts` 页面识别、卡片扫描、日期补全
- `TikTok` 适配器负责账号主页视频列表识别、卡片扫描、日期解析

当前落地状态：

- YouTube：链接采集、日期展示、导出已稳定接通
- TikTok：链接采集、数据入库、导出已接通
- TikTok：日期解析逻辑已接入，但卡片下方的日期标签展示仍需继续适配页面结构

### 3.2 popup UI

文件：

- `/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension/popup.html`
- `/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension/popup.css`
- `/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension/popup.js`

职责：

- 渲染当前页面状态
- 触发“采集当前可见封面”
- 展示累计数量、批次号、最后断点
- 调用下载能力导出 `.xls` 和 `.json`
- 支持清空当前页面进度

### 3.3 本地存储

使用：`chrome.storage.local`

按 `平台 + 页面地址 + 排序方式` 作为上下文键保存数据，避免不同站点、不同频道、不同排序之间相互污染。

示意结构：

```json
{
  "version": 1,
  "datasets": {
    "https://www.youtube.com/@NBAACTION/shorts::最早": {
      "pageMeta": {},
      "batchNo": 3,
      "lastScanAt": "2026-03-18T09:00:00.000Z",
      "lastCheckpoint": {},
      "scanHistory": [],
      "items": []
    }
  }
}
```

## 4. 去重与断点策略

### 4.1 去重

以以下优先顺序做唯一键：

1. `videoId`
2. `url`

当再次扫描到同一个视频时：

- 不重复新增记录
- 只更新最近一次看到它的时间和批次

### 4.2 断点

每次采集后，记录当前批次最后一个封面：

- 标题
- 链接
- 视频 ID
- 缩略图链接
- 批次号
- 屏内顺序
- 采集时间

这样用户下次继续时，可以先根据标题或链接定位，再从后面开始。

## 5. 发布时间获取策略

发布时间采用两层解析策略：

YouTube：

1. 先尝试直接从 Shorts 列表卡片中提取日期信息。
2. 如果列表卡片里没有精确日期，再请求对应视频页，从页面元数据中读取 `datePublished`。

TikTok：

1. 优先读取主页已渲染状态中的日期信息。
2. 如果主页状态中没有直接给出日期，再根据视频 ID 还原创建时间对应日期。

这样 TikTok 默认不需要逐条访问每个视频详情页，可明显降低短时间内额外请求数量。

最终统一格式化为 `YYYY-MM-DD`，写入导出表格。

## 6. 导出方案

### 5.1 Excel

当前导出为 Excel 兼容 `.xls` 文件，本质是 HTML 表格包装格式。

优点：

- 不依赖第三方库
- 浏览器直接可生成
- Excel 可以直接打开

### 5.2 JSON

同时支持导出 `.json`，作用是：

- 备份浏览器中的本地进度
- 后续若要升级到 `.xlsx`、数据库或自动化流程，JSON 可直接复用

## 7. 当前限制

- 仅支持手动滚动 + 手动点击采集
- 仅支持页面中已渲染并可见的视频卡片
- YouTube 发布时间仍可能受页面元数据可用性影响
- TikTok 日期优先依赖主页状态或视频 ID 解码，若平台规则大改需要同步调整
- TikTok 日期标签目前仍存在“已解析但未稳定显示在卡片下方”的适配问题
- 页面结构变化时，需要同步调整选择器
- 暂未保存“最后一个封面截图”

## 8. 后续迭代建议

### 7.1 自动滚动采集

下一阶段可以增加：

- 自动滚动
- 检测新卡片加载
- 自动分批采集
- 达到底部后自动停止

### 7.2 原生 `.xlsx`

可选方案：

- 引入前端 `xlsx` 类库
- 或补一个本地 Python 导出脚本

### 7.3 更强的断点恢复

可以把以下信息一并保存：

- 最后一个封面截图
- 页面滚动位置
- 最后一条记录的 DOM 位置信息

这样恢复时会更直观。
