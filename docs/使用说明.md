# 使用说明

## 1. 安装扩展

1. 打开 Chrome 浏览器。
2. 地址栏输入 `chrome://extensions/` 并回车。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择目录：
   `/Users/apple/Desktop/开发项目/获取youtube页面链接/getYoutubeLink/chrome-extension`
6. 加载成功后，把扩展固定到工具栏，方便后续点击。

## 2. 推荐使用流程

### 第一步：打开目标页面

打开目标页面。

YouTube 示例：

- `https://www.youtube.com/@NBAACTION/shorts`

TikTok 示例：

- `https://www.tiktok.com/@lifehacksor`

然后根据你的业务需求，选择页面里的排序方式，例如：

- 最新
- 最热门 / 最热
- 最早

建议一次只采集一种排序方式，便于后续整理。

补充说明：

- 打开 YouTube Shorts 列表后，扩展会自动在每个视频卡片下方补一行 `发布时间：YYYY-MM-DD`
- YouTube 某些卡片的日期可能会晚一两秒出现，这是插件在读取视频页元数据
- TikTok 页面默认不会逐条访问每个视频详情页，而是优先从主页状态或视频 ID 还原日期
- TikTok 当前已经支持链接采集和导出，但日期标签在视频封面下方的显示还在继续适配，部分页面暂时可能看不到

### 第二步：采集当前可见封面

1. 停留在当前可见的一屏视频封面。
2. 点击扩展图标。
3. 点击 `采集当前可见封面`。
4. 扩展会自动完成：
   - 提取当前可见封面的链接
   - 补充每条视频的发布时间
   - 自动去重
   - 更新累计数量
   - 记录本次批次
   - 记录当前最后一个封面作为断点

### 第三步：继续往下滚动

1. 页面向下滚动一屏或几屏。
2. 等页面封面加载出来。
3. 再次点击 `采集当前可见封面`。
4. 重复此流程，直到你采集够了为止。

## 3. 如何知道上次采集到哪里

扩展弹窗里会显示：

- 最近一次采集时间
- 当前累计采集数
- 当前批次号
- `上次停在` 的最后一个封面标题
- 最后一个封面的链接

这意味着：

- 下次继续前，你只要先滚动到接近该位置
- 对照标题或链接确认
- 然后从它后面继续采集

## 4. 如何导出 Excel 表格

1. 点击扩展图标。
2. 点击 `导出 Excel 兼容表格`。
3. 浏览器会下载一个 `.xls` 文件。
4. 该文件可直接使用 Excel 打开。

说明：

- 当前导出的是 Excel 兼容格式，便于直接查看和交付。
- `发布时间` 会以 `YYYY-MM-DD` 的形式写入表格。
- 如果你还想做备份，可以同时点击 `导出 JSON 备份`。

## 5. 如何清空当前页面进度

如果你想重新开始采集某个页面：

1. 打开同一个页面上下文。
2. 点击扩展图标。
3. 点击 `清空当前页面进度`。
4. 确认后，该页面当前排序方式下的记录会被清空。

注意：

- 清空的是浏览器本地进度。
- 已经导出的 `.xls` 或 `.json` 文件不会被删除。

## 6. 建议的操作习惯

为了让断点更清晰，建议你这样用：

- 每次固定使用同一种排序方式完成一轮采集
- 每采集一到两屏就执行一次采集，不要滚太多再采
- 每轮结束都导出一次 Excel 和 JSON
- 如果要多频道采集，尽量一频道一轮，不要来回切太多

## 7. 常见问题

### 7.1 点击采集后没有数据

请检查：

- 当前页面是否真的是 YouTube 频道的 `Shorts` 页面，或 TikTok 账号主页视频列表
- 当前封面是否已经加载出来
- 是否是刚安装扩展但页面还没刷新

如果是刚安装扩展，建议刷新当前页面后再试一次。

### 7.2 为什么总数没有增加

可能原因：

- 当前这一屏内容之前已经采集过
- 扩展正在按 `视频ID / 链接` 去重

这是正常行为，说明重复内容没有被再次计入。

### 7.3 为什么不是 `.xlsx`

当前版本优先保证：

- 安装简单
- 无需额外依赖
- 可直接用 Excel 打开

所以先采用了 Excel 兼容 `.xls` 导出方式。后续版本可以再升级到原生 `.xlsx`。
